# Deployment Plan for Azure

This plan outlines the steps to deploy your Dockerized .NET application and PostgreSQL database to Azure using **Azure Container Apps** or **App Service for Containers**. We will focus on the most straightforward path for a beginner.

## 1. Create a Docker Hub Account & Repository
To deploy to Azure, your Docker image needs to be stored in a registry that Azure can access.
- **Action**: You will need to sign up for a free Docker Hub account (if you don't have one).
- **Action**: Create a public (or private) repository named `tasktracker`.

## 2. Push Image to Docker Hub
We will use the Docker CLI to build and push your image.
- **Login**: Run `docker login` in your terminal.
- **Tag**: Tag your local image with your Docker Hub username: `docker tag test4ds-web <your-username>/tasktracker:latest`.
- **Push**: Upload the image: `docker push <your-username>/tasktracker:latest`.

## 3. Create Azure Resources
We will use the Azure CLI (or Portal) to set up the infrastructure. Since you are new, we will generate a script using Azure CLI commands that you can run.
- **Resource Group**: A container for all resources.
- **PostgreSQL Database**: Azure Database for PostgreSQL (Flexible Server).
- **Container App / App Service**: The service that runs your Docker image.

## 4. Configure Environment Variables
We need to tell the Azure app how to connect to the Azure database.
- **Connection String**: We will construct the production connection string using the Azure PostgreSQL credentials.
- **App Settings**: Set `ConnectionStrings__DefaultConnection` in the Azure App Service configuration.

## 5. Deployment
- Once the App Service is created and pointed to your Docker Hub image, Azure will pull the image and start the container.
- We will verify the deployment by accessing the Azure URL.

**Note**: This process involves creating real cloud resources which may incur costs. Please ensure you have an active Azure subscription.
