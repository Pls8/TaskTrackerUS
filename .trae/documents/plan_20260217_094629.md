# Docker Deployment Plan

This plan will containerize your application and database, allowing you to run the entire system with a single command (`docker-compose up`).

## 1. Create `Dockerfile`
We will create a `Dockerfile` in the solution root to build the .NET application image.
- **Base Image**: Use the official .NET SDK image to build and the ASP.NET Core runtime image to run.
- **Build Steps**: Restore dependencies, build the project, and publish the output.
- **Entrypoint**: Configure the container to start `TaskTracker.Web.dll`.

## 2. Create `docker-compose.yml`
We will create a `docker-compose.yml` file to orchestrate the services.
- **Service: `db`**:
  - Image: `postgres:latest`
  - Environment: Set `POSTGRES_USER`, `POSTGRES_PASSWORD`, and `POSTGRES_DB`.
  - Ports: Map port 5432 (optional, for local access).
  - Volumes: Persist database data so it survives container restarts.
- **Service: `web`**:
  - Build: Use the `Dockerfile` from step 1.
  - Ports: Map port 8080 (container) to 5000 (host).
  - Environment: Override the `ConnectionStrings__DefaultConnection` to point to the `db` service instead of `localhost`.
  - Depends On: Ensure `db` starts before `web`.

## 3. Update Application Configuration
- The application already applies migrations on startup (`Program.cs`), so the database will be automatically created and seeded when the container starts.
- We will verify that the connection string in `docker-compose.yml` matches the credentials set for the `db` service.

## 4. Execution
- Run `docker-compose up --build` to build the images and start the containers.
- Access the application at `http://localhost:5000`.
