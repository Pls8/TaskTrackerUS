# Deployment Fix Plan: Render Connection Refused

The error `Npgsql.NpgsqlException (0x80004005): Failed to connect to 127.0.0.1:5432` confirms that the application is trying to connect to **localhost (127.0.0.1)** instead of the remote PostgreSQL server provided by Render.

## Root Cause
- The application is falling back to the default connection string in `appsettings.json` (which points to localhost) because it is failing to read or correctly parse the `DATABASE_URL` environment variable provided by Render.
- This suggests that either:
    1.  The environment variable name on Render is different (e.g., `ConnectionString` vs `DATABASE_URL`).
    2.  The logic I added to `DependencyInjection.cs` to parse the connection string is not being triggered, or `configuration.GetConnectionString("DefaultConnection")` is returning null/empty, or it's not picking up the environment variable override.

## Solution
We need to robustly read the connection string from the environment variable that Render provides. Render typically uses `DATABASE_URL`.

### 1. Update `DependencyInjection.cs`
I will modify the code to explicitly check for the `DATABASE_URL` environment variable if the configuration doesn't provide a valid cloud connection string.

**Proposed Logic:**
1.  Try to get `DATABASE_URL` from environment variables directly.
2.  If found, use that (and parse it if it's a URI).
3.  If not found, fall back to `configuration.GetConnectionString("DefaultConnection")`.

### 2. Verify Environment Variable Name
(Self-Correction/Check): Render's PostgreSQL service provides an "Internal Connection String" (usually `postgres://...`) and an "External Connection String". The environment variable is often named `DATABASE_URL` automatically if you link the services, or you might have manually added it as `ConnectionStrings__DefaultConnection`.

Given the error, it's safest to check **both**.

### 3. Rebuild and Push
After patching the code, we will rebuild and push the Docker image again.

I will start by modifying `TaskTracker.Infrastructure/DependencyInjection.cs` to prioritize the environment variable `DATABASE_URL`.
